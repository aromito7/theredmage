Project Euler

Problem 051

By replacing the 1st digit of the 2-digit number *3, it turns out that six of the nine possible values: 13, 23, 43, 53, 73, and 83, are all prime.

By replacing the 3rd and 4th digits of 56**3 with the same digit, this 5-digit number is the first example having seven primes among the ten generated numbers, yielding the family: 56003, 56113, 56333, 56443, 56663, 56773, and 56993. Consequently 56003, being the first member of this family, is the smallest prime with this property.

Find the smallest prime which, by replacing part of the number (not necessarily adjacent digits) with the same digit, is part of an eight prime value family.

The first attempt

My first instinct was to initialize an array full of zeros with the NumPy to count all of the number families that use '*' in them. I would iterate through all numbers below 1 million checking to see if they are prime. If they are prime, I increase the count for all possible families they could belong to.

Take 56003 for example, after confirming that it's prime, we can count it in any number of prime families that it is part of by substiting any non-zero subset of digits with a single value value with a *. We generate these families by replacing 5, 6, 3, or any combination of 0s to find it belongs to six families: 5600*, 5*003, *6003, 56**3, 560*3, 56*03. We would then increase the count of each of the corresponding values in our array. Each dimension of the array would have size 11 where 0 to 9 represent themselves in the array and 10 represents the *. To increase the count for family 56**3 the code would look something like "nparray[5][6][10][10][3]+=1".

However, simply creating an array of size 11x11x11x11x11x11 for six digits required more than a few seconds for my computer to do, and I decided that working with a sparse array of this size was not an efficient way to go about solving the problem.
